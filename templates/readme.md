# Consulting Templates

This folder contains reusable documentation templates used in architecture design, technical delivery, and client discovery engagements.

These templates are aligned with the Architecture Decision Record (ADR) structure in this repository and emphasize:

- Explicit decision drivers
- Clear trade-offs
- Security and compliance awareness
- Failure-mode thinking
- Validation and revisit criteria
- Practical, delivery-oriented clarity

---

## Templates Included

### 1. ADR Template

A standardized format for documenting significant architectural decisions.

Use when:
- Choosing infrastructure platforms
- Defining data models
- Establishing tenant isolation strategy
- Selecting architectural patterns
- Making decisions with long-term impact

---

### 2. Technical Design Document Template

A structured format for describing system-level or feature-level design.

Use when:
- Designing new subsystems
- Introducing new integrations
- Performing architectural modernization
- Planning complex feature rollouts

This template integrates:
- Security considerations
- Failure-mode analysis
- Observability planning
- Trade-off documentation

---

### 3. Discovery Workshop Template

A facilitation guide for client-facing discovery sessions.

Use when:
- Modernizing legacy systems
- Designing new SaaS platforms
- Evaluating AI enablement feasibility
- Assessing scalability, reliability, or compliance constraints

The goal is to translate ambiguous business problems into structured architectural direction.

---

## Philosophy

Documentation in this repository is intentionally:

- Lightweight but rigorous
- Security-aware
- Trade-off explicit
- Revisit-able as constraints evolve
- Focused on delivery, not bureaucracy
