# Consulting Templates

This folder contains reusable documentation templates used during architecture, delivery planning, and client discovery engagements.

These templates are designed to balance speed with clarity — enabling structured thinking without introducing unnecessary documentation overhead.

---

## Templates Included

### 1. ADR Template
A lightweight Architecture Decision Record format for documenting significant technical decisions.

Use when:
- Selecting infrastructure components
- Choosing architectural patterns
- Defining long-term technical direction


### 2. Technical Design Document Template
A structured format for describing feature-level or system-level design.

Use when:
- Designing new subsystems
- Introducing new integrations
- Planning significant feature implementations


### 3. Discovery Workshop Template
A structured facilitation guide for client-facing architecture discovery sessions.

Use when:
- Modernizing legacy systems
- Designing new SaaS platforms
- Assessing scalability and reliability concerns
- Evaluating AI enablement feasibility

---

## Philosophy

Good documentation should:

- Reduce ambiguity
- Surface assumptions
- Clarify trade-offs
- Enable independent execution
- Support stakeholder alignment

The goal is not heavy process — it is architectural confidence and delivery acceleration.
